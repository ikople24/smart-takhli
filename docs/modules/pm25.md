# PM2.5 (DustBoy)

แสดงค่าฝุ่น PM2.5 หน้า public — อ่านจาก **Mongo cache เท่านั้น**
DustBoy API ถูกเรียกเฉพาะใน cron

## หน้า / Components

- หน้า public/home แสดงค่าจาก cache, `components/Pmdata.js` (⚠️ root — รอเฟส 5)
- `/admin/pm25-settings` — สลับโหมดแหล่งข้อมูล (Sheet / DustBoy API) + ปุ่ม
  Export CSV รายงานรายเดือน (`components/pm25/ExportPm25Monthly.jsx` —
  โฟลเดอร์แรกของโมดูลนี้ใน `components/`)

## API / Cron

- `pages/api/pm25/*` (มี `_auth.js` ของตัวเอง)
- `GET /api/pm25/monthly-report` — ทุกเดือนจาก `Pm25Monthly` (เรียงเก่า→ใหม่)
  สำหรับปุ่ม export CSV บนหน้า pm25-settings (ป้องกันด้วย `requirePm25Admin`)
- Cron (อย่าใส่ Clerk — ใช้ `requireCronSecret` จาก `lib/pm25CronAuth.js`):
  - `5 * * * *` รายชั่วโมง · `15 0 * * *` daily ย้อน 7 วัน · `0 3 * * *` monthly
    (self-healing: upsert ทุกเดือนจาก data1year ทุกรอบ เติมเดือนที่ cron พลาด;
    เดือนปัจจุบันอัปเดตทุกวัน เดือนที่จบแล้วเขียนซ้ำเฉพาะ count เพิ่ม —
    เรียก data1year วันละ 1 ครั้ง (จาก ~เดือนละครั้งเดิม); ถ้า DustBoy แก้ข้อมูล
    ย้อนหลังแบบ count ลดลง ระบบจะไม่ทับให้ ต้องแก้ตรงที่ DB/script เอง)
  - เรียกจาก scheduler ภายนอกด้วย header `x-cron-secret` หรือ `?secret=`

## เครื่องวัดของเทศบาล (ESP8266) — ส่งค่าเข้า Mongo ตรง

นอกจาก DustBoy ยังมีเครื่องวัดของเทศบาลเอง (ESP8266 + DHT11 + PMSx003 วัด
temp/humidity/pm1/pm2.5/pm10) เดิมส่งผ่าน Google Apps Script → Google Sheet
ย้ายมาส่งเข้า Mongo ตรงแทน:

- `POST /api/pm25/device-ingest` — public endpoint (machine client ไม่มี Clerk
  session) ป้องกันด้วย shared secret ใน header `x-device-secret` เทียบกับ
  `PM25_DEVICE_SECRET` (เลียนแบบ `requireCronSecret`) → insert เป็น time-series
- 1 reading = 1 document ใน collection `pm25_device_readings` (model
  `Pm25DeviceReading`) — แยกจาก cache ของ DustBoy ทั้งหมด ไม่ทับ `Pm25Latest`
- รับ field: `temp, humidity (หรือ humi), pm1, pm25, pm10, deviceId?` — ต้องมี
  ค่าวัดอย่างน้อยหนึ่งตัว ไม่งั้น 400
- ⚠️ ยังไม่ผูกกับหน้า public/home (ของเดิมอ่าน DustBoy) — ถ้าจะแสดงค่าเครื่องนี้
  ต้องทำหน้า/วิดเจ็ตอ่านจาก `pm25_device_readings` เพิ่ม

## Models

`Pm25Latest`, `Pm25DailySnapshot`, `Pm25Monthly`, `Pm25Settings`, `Pm25SyncLog`,
`Pm25DeviceReading` (เครื่องวัดเทศบาล — collection `pm25_device_readings`)

## Lib

`lib/pm25Sync.js` (sync logic), `lib/pm25Data.js` (อ่าน cache),
`lib/pm25CronAuth.js` (`requireCronSecret` สำหรับ cron),
`pages/api/pm25/_auth.js` — มี `requirePm25Admin` ซึ่งเป็น**ต้นแบบ auth มาตรฐาน**
ของ API ฝั่ง admin ทั้งโปรเจกต์ (getAuth → Mongo lookup → appId → allowedPages → superadmin)

## Env vars

`DUSTBOY_API_KEY`, `DUSTBOY_STATION_ID`, `DUSTBOY_API_BASE`, `CRON_SECRET`,
`PM25_DEVICE_SECRET` (shared secret ของเครื่องวัด ESP8266)
