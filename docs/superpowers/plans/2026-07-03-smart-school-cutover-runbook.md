# Smart School — Cutover Runbook (รันตอน deploy จริง)

โค้ดทั้งหมดของโมดูล `smart-school` เสร็จและรีวิวแล้วบน branch `smart-school`
(build + lint ผ่าน). ขั้นตอนด้านล่างนี้เป็น **การตัดข้ามระบบจริง (cutover)** —
เขียนข้อมูลลงฐานโปรดักชันและส่งแจ้งเตือน Telegram จริง จึง **ต้องรันโดยผู้ใช้
ตอน deploy** ไม่ใช่ระหว่างพัฒนา

n8n workflow "Api All" **แก้และ publish แล้ว** (active version
`1fe9305f-39a3-4c79-84e3-ee3b15f5240c`) — ข้อ 4 ด้านล่างจึงพร้อมรับ payload ใหม่ทันที

## ลำดับที่แนะนำ

### 1) Merge + deploy branch `smart-school`
เปิด PR เข้า main แล้ว deploy ตามปกติ (โค้ดใหม่อ่าน collection ใหม่;
ของเดิม `educationregisters` ไม่ถูกแตะ) — migration ควรรัน **หลัง/พร้อม** deploy
เพื่อไม่ให้ collection ใหม่ค้างว่างระหว่างที่ยังไม่มีโค้ดอ่าน

### 2) Migration ข้อมูลเก่า 238 รายการ (idempotent)
```bash
node --env-file=.env.local scripts/migrate-education-to-smart-school.js --dry-run
node --env-file=.env.local scripts/migrate-education-to-smart-school.js
```
คาดหวัง: `สร้างใหม่ 238 / เติมใบสมัครที่ขาด 0 / ข้ามที่ย้ายแล้ว 0` แล้วบรรทัดยอด
`applicants=238, applications(2568)=238`. รันซ้ำได้ (ครั้งถัดไปได้ `สร้างใหม่ 0 /
ข้าม 238`). ถ้ายอด applications < 238 สคริปต์จะ exit code 1 พร้อมข้อความให้รันซ้ำ

### 3) ให้สิทธิ์หน้าใหม่แก่แอดมินเดิม
```bash
node --env-file=.env.local scripts/grant-smart-school-permission.js --dry-run
node --env-file=.env.local scripts/grant-smart-school-permission.js
```
(dry-run แสดง 3 ราย ณ วันเขียน: ธนวัฒน์, สรรเสริญ, Thanawat) — เพิ่ม
`/admin/smart-school` ให้ user ที่เคยมี `/admin/education-map`

### 4) ⚠️ ตั้งฐานกติกาหมุนเวียนทุน (สำคัญ — ต้องทำก่อนเริ่มพิจารณาปี 2569)
migration ตั้งทุกรายการปี 2568 เป็นสถานะ `ตรวจสอบแล้ว` — **ไม่มีรายการไหนเป็น
`ได้รับทุน`** เพราะข้อมูลเก่าไม่ได้บันทึกผลว่าใครได้ทุนจริงในปี 2568
(ช่อง `takhliScholarshipHistory` เป็น self-report ของปี 2565–2567 เท่านั้น
เชื่อถือเป็นฐานไม่ได้)

ผลคือ ป้ายเตือน "🔁 ได้ทุนปีที่แล้ว — ต้องหมุนเวียนเปลี่ยนคน" ในปี 2569
**จะไม่ขึ้น** จนกว่าเจ้าหน้าที่จะระบุผู้ที่ได้ทุนปี 2568 เข้าระบบก่อน

**สิ่งที่ต้องทำ:** เจ้าหน้าที่ (ที่มีรายชื่อผู้ได้ทุนปี 2568) เปิด
`/admin/smart-school` แท็บปี **2568** แล้วเปลี่ยนสถานะผู้ที่ได้ทุนจริงเป็น
`ได้รับทุน` — จากนั้นระบบจะเตือนกติกาครัวเรือน/หมุนเวียนให้อัตโนมัติในปี 2569
(ถ้ามีรายชื่อเป็นไฟล์ ทำสคริปต์ bulk ให้เพิ่มได้ แต่ต้องมีข้อมูลจากเทศบาลก่อน)

### 5) ตรวจรับฟอร์มสาธารณะ (หน้าแรก) — 4 เส้นทาง
ใช้ชื่อทดสอบ "ทดสอบ ระบบ" แล้วลบทิ้งภายหลังผ่านหน้าแอดมิน. เลขบัตรทดสอบที่
checksum ถูกต้อง: `1112034563562` (สร้างเพิ่มได้ตามต้องการ)
1. **รายใหม่:** เลข valid ใหม่ → กรอก → ส่ง → Swal ขึ้น `TKC69-001` + Telegram "📚 รายใหม่"
2. **รายเก่าเจอด้วยเลขบัตร:** ใช้เลขเดิมจากข้อ 1 → เลือกรายเก่า → ยืนยัน "ใช่ฉัน" →
   ฟอร์ม prefill → แก้รายได้ → ส่ง → Telegram "🔄 รายเก่า update ข้อมูล" (ใบเดิมถูก update ไม่สร้างซ้ำ)
3. **รายเก่าค้นชื่อ+ผูกเลข:** ค้นชื่อจากข้อมูล migrate (เช่น "ณรงค์ฤทธิ์") → เลือก →
   กรอกเลข valid → ผูกเลขสำเร็จ + prefill ปี 2568 → **ไม่ต้องกดส่ง** → ตรวจในแอดมินว่า
   applicant มี citizenId แล้ว และค้นชื่อเดิมซ้ำต้อง **ไม่เจอ** (ผูกแล้วหายจากค้นชื่อ)
4. **หาไม่เจอ→รายใหม่:** ค้นชื่อมั่ว → ไม่เจอ → ปุ่ม "ยื่นเป็นรายใหม่" ใช้ได้

### 6) ตรวจรับหน้าแอดมิน `/admin/smart-school`
- แท็บปี 2568 เห็นข้อมูล migrate ครบ; แท็บ 2569 เห็นรายการทดสอบ
- แก้รูปในรายการทดสอบ → บันทึกแล้วรูปเปลี่ยนจริง + Telegram "🖼️ เปลี่ยนรูปภาพ"
- สร้าง 2 ใบเบอร์เดียวกัน → เห็นป้าย "🏠 บ้านเดียวกัน"; ตั้ง "ได้รับทุน" ใบแรกแล้ว
  ลองตั้งใบที่สอง → มี dialog เตือน
- ลบรายการทดสอบทั้งหมด (ปุ่มลบในหน้า detail) — applicant ที่ไม่เหลือใบถูกลบตาม
- `/admin/education-map` เด้ง redirect ไป `/admin/smart-school`

### 7) ตรวจรับสุดท้าย
```bash
npm run lint && npm run build
```
ทั้งคู่ต้องผ่าน (ยืนยันแล้วบน branch — warning `useContext` ของ Clerk เป็นของเดิม ไม่เกี่ยว)

## หมายเหตุความปลอดภัยข้อมูล
- collection เดิม `educationregisters` = **backup ถาวร** ห้ามลบ/แก้
- migration ยึด `legacyId` — ปลอดภัยต่อการรันซ้ำ, กันตายกลางคัน (เติมใบที่ขาดให้)
- ปิดแจ้งเตือน n8n ชั่วคราวได้ด้วย env `N8N_SCHOOL_WEBHOOK_URL=""` (เช่นตอนทดสอบ
  ไม่อยากรบกวนกลุ่ม Telegram)
